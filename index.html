Authentication

Coinley uses API key authentication. Include your API key and secret in the request headers.

Headers Required

X-API-Key: your_api_key_here
X-API-Secret: your_api_secret_here
Content-Type: application/json

Important

Keep your API credentials secure. Never expose them in client-side code.

Create Payment

Create a new payment request to receive stablecoins from your customers.

POST /api/payments/create

Request Parameters

Parameter Type Required Description
amount string Required Payment amount (e.g., "100.00")
currency string Required Currency symbol (e.g., "USDT", "USDC")
network string Required Network name (ethereum, polygon, bsc, etc.)
customerEmail string Optional Customer's email address
orderId string Optional Your internal order ID
callbackUrl string Optional Webhook URL for payment notifications

Example Request

curl -X POST https://api.coinley.com/api/payments/create \
  -H "X-API-Key: your_api_key" \
  -H "X-API-Secret: your_api_secret" \
  -H "Content-Type: application/json" \
  -d '{
    "amount": "100.00",
    "currency": "USDT",
    "network": "ethereum",
    "customerEmail": "customer@example.com",
    "orderId": "order_12345",
    "callbackUrl": "https://yoursite.com/webhook"
  }'

Process Payment

Verify and process a payment using the transaction hash from the blockchain.

POST /api/payments/process

Request Parameters

Parameter Type Required Description
paymentId string Required Payment ID from create payment response
transactionHash string Required Transaction hash from blockchain
network string Required Network where transaction was made
senderAddress string Optional Customer's wallet address

Example Request

curl -X POST https://api.coinley.com/api/payments/process \
  -H "X-API-Key: your_api_key" \
  -H "X-API-Secret: your_api_secret" \
  -H "Content-Type: application/json" \
  -d '{
    "paymentId": "uuid-payment-id",
    "transactionHash": "0xabcdef1234567890...",
    "network": "ethereum",
    "senderAddress": "0x9876...abcd"
  }'

Example Response

{
  "success": true,
  "message": "Payment processed successfully",
  "payment": {
    "id": "uuid-payment-id",
    "status": "completed",
    "amount": 100.00,
    "transactionHash": "0xabcdef1234567890...",
    "senderAddress": "0x9876...abcd"
  },
  "onChainData": {
    "blockNumber": 18745123,
    "gasUsed": "65432",
    "confirmed": true
  },
  "verificationAttempts": 1
}

Webhooks

Receive real-time notifications when payment statuses change.

Webhook Events

  • payment.completed - Payment successfully processed
  • payment.failed - Payment verification failed
  • payment.expired - Payment expired without completion

Webhook Payload Example

{
  "event": "payment.completed",
  "timestamp": "2024-01-15T10:45:00Z",
  "data": {
    "paymentId": "uuid-payment-id",
    "amount": 100.00,
    "currency": "USDT",
    "network": "ethereum",
    "status": "completed",
    "transactionHash": "0xabcdef...",
    "merchantId": "uuid-merchant-id",
    "orderId": "order_12345"
  },
  "signature": "sha256_webhook_signature"
}

Webhook Security

Always verify webhook signatures using your webhook secret to ensure authenticity.

Error Handling

Coinley uses conventional HTTP response codes to indicate success or failure.

Success Codes

  • 200 - OK
  • 201 - Created

Error Codes

  • 400 - Bad Request
  • 401 - Unauthorized
  • 404 - Not Found
  • 500 - Internal Server Error

Error Response Format

{
  "success": false,
  "error": "Validation failed",
  "message": "Amount is required and must be a positive number",
  "code": "VALIDATION_ERROR",
  "timestamp": "2024-01-15T10:30:00Z"
}
